Plan

The review is conducted using a shared GitHub repository, which was be synchronized locally by the team

Record metadata is curated as follows:

  • Data retrieved in the search is stored in the data/search directory; the Git history of this path shows that the files were preserved in their original form, i.e., treated as raw data
  • Records were imported into the data/records.bib as the primary data structure; the Git history of this file documents how each record evolved through the process (e.g., manual or computational change of metadata, merging of records, prescreening decisions)

For primary data (record metadata), the Bibtex format was chosen and consistent formatting was ensured by CoLRev (Wagner and Prester 2025). BibTex is machine readable and the changes can easily be interpreted when inspecting the git history.

Search

The search strategie are stored in JSON format together with the raw data files in the data/search directory, in line with the standard of Haddaway et al. (2022). For the illustration, we relied on open-access API-searches, because licensing issues permit publication of raw data exported from databases like WOS or EBSCO. We also reused samples from prior reviews, as outlined in Table 1.
